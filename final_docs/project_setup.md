## 3. Backend Setup (FastAPI)

### Step 1 — Create the backend folder and virtual environment
```bash
mkdir backend
cd backend
python -m venv venv
```

Activate it:
```bash
# Windows
venv\Scripts\activate

# Mac/Linux
source venv/bin/activate
```

### Step 2 — Install core packages
```bash
pip install fastapi uvicorn[standard] sqlalchemy alembic psycopg2-binary pydantic-settings python-jose[cryptography] passlib[bcrypt] python-multipart
```

| Package | Why we need it |
|---|---|
| fastapi | The web framework itself |
| uvicorn | Runs the FastAPI server |
| sqlalchemy | Talks to the database (ORM) |
| alembic | Manages database migrations |
| psycopg2-binary | PostgreSQL driver |
| pydantic-settings | Loads config from `.env` |
| python-jose | Creates and verifies login tokens (JWT) |
| passlib[bcrypt] | Hashes passwords safely |
| python-multipart | Needed for file uploads |

Save the installed versions:
```bash
pip freeze > requirements.txt
```

### Step 3 — Create the app folder structure
```bash
mkdir app app\api app\models app\schemas app\services app\db app\core
```

Create empty starter files:
- `app/main.py` — creates the FastAPI app and includes all routers
- `app/core/config.py` — reads settings from `.env` (database URL, secret key, etc.)
- `app/db/session.py` — sets up the database connection
- `app/models/*.py` — one file per table group (e.g. `product.py`, `order.py`)
- `app/schemas/*.py` — request/response models, matching `api_reference.md`
- `app/api/*.py` — one router file per domain (e.g. `products.py`, `orders.py`)
- `app/services/*.py` — business logic (GST calculation, stock movement, etc.)

### Step 4 — Set up environment variables
Create `backend/.env`:
```env
DATABASE_URL=postgresql://username:password@localhost:5432/dms_db
SECRET_KEY=change-this-to-a-random-secret
ACCESS_TOKEN_EXPIRE_MINUTES=60
```

### Step 5 — Create the database
```bash
psql -U postgres
CREATE DATABASE dms_db;
```

### Step 6 — Set up Alembic (migrations)
```bash
alembic init alembic
```
Then update `alembic.ini` and `alembic/env.py` to point to `DATABASE_URL` from `.env`, and to your SQLAlchemy models.

Build tables to match `database_schema_docs.markdown`, then generate and run the first migration:
```bash
alembic revision --autogenerate -m "initial schema"
alembic upgrade head
```

### Step 7 — Run the backend
```bash
uvicorn app.main:app --reload
```
Backend runs at: `http://localhost:8000`
Interactive API docs (auto-generated by FastAPI): `http://localhost:8000/docs`

## 4. Frontend Setup (Next.js)

### Step 1 — Create the Next.js project
From the project root:
```bash
npx create-next-app@latest frontend
```
When prompted, choose:
- TypeScript: **Yes**
- ESLint: **Yes**
- Tailwind CSS: **Yes** (or your preferred styling tool)
- App Router: **Yes**
- `src/` directory: optional, either is fine

### Step 2 — Install extra packages
```bash
cd frontend
npm install axios
```
`axios` (or `fetch`) is used to call the FastAPI backend.

### Step 3 — Set up environment variables
Create `frontend/.env.local`:
```env
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000/api/v1
```

### Step 4 — Create the API client
Create `frontend/lib/api.ts` — a single place that wraps calls to the backend using `NEXT_PUBLIC_API_BASE_URL`, so every page/component reuses it instead of hardcoding URLs.

### Step 5 — Create TypeScript types matching the API
In `frontend/types/`, create one type file per domain (e.g. `order.ts`, `product.ts`) matching the request/response shapes described in `api_reference.md`. This keeps frontend and backend in sync.

### Step 6 — Run the frontend
```bash
npm run dev
```
Frontend runs at: `http://localhost:3000`

## 5. Connecting Frontend to Backend

- The frontend calls the backend using the base URL from `NEXT_PUBLIC_API_BASE_URL`.
- During local development, allow the frontend's origin in the backend's CORS settings (in `app/main.py`):
```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```
- All API routes follow the `/api/v1/...` structure defined in `apis_doc.md`.

## 6. Suggested Build Order

Follow this order so each piece has what it depends on already in place:

1. Database schema (tables from `database_schema_docs.markdown`) + Alembic migration.
2. Auth & Users APIs (needed to log in and test everything else).
3. Master data APIs — Routes, Customers, Categories, Brands, Products, Price Lists, Warehouses, Suppliers, Vehicles.
4. Selling flow — Orders, Invoices, Deliveries, Payments.
5. Buying flow — Purchases.
6. Returns flow.
7. Inventory ledger + Audit log (wired into the flows above, not built standalone).
8. File uploads.
9. Tally sync.
10. Frontend pages, screen by screen, matching the same order.

This matches the phased rollout in `prd.md` (Section 11 — Milestones).

## 7. Daily Development Checklist

- Pull latest code: `git pull`
- Backend: activate virtual environment, run `alembic upgrade head` if there are new migrations, then `uvicorn app.main:app --reload`
- Frontend: `npm run dev`
- Check API docs at `http://localhost:8000/docs` when unsure of a request/response shape
- Cross-check new endpoints against `api_reference.md` before building the matching frontend screen

## 8. Common Issues

| Problem | Likely Cause | Fix |
|---|---|---|
| `uvicorn` command not found | Virtual environment not activated | Run the `activate` command again |
| Database connection error | Wrong `DATABASE_URL` or PostgreSQL not running | Check `.env` and confirm PostgreSQL service is running |
| CORS error in browser | Frontend origin not allowed in backend | Add frontend URL to `CORSMiddleware` in `main.py` |
| Frontend calls fail with 404 | Wrong `NEXT_PUBLIC_API_BASE_URL` or missing `/api/v1` prefix | Check `.env.local` |
| Migration conflicts | Two people changed models and generated migrations separately | Pull latest, regenerate migration on top of the latest one |
